You can download this font here. Also, you can download a smaller font from the previous example here.

As you may remember, the width of the space in the font "medium" is equal to 4 symbols. In the font "roman" your program should set the width of the space from the symbol "a" - space should be 10 symbols wide.

Pretty accurate, isn't it? Unfortunately, there is no space to include all lowercase and all uppercase letters.

All these fonts have a special description, included in their .txt file. The first line of the file contains two numbers. The first number is font size, indicating how many lines are needed to print one line of text using this font. For example, the font "medium" needs 3 lines, and the font "roman" needs 10. The second number is the number of characters described in the file. Both fonts have information about 52 characters: 26 uppercase and 26 lowercase letters.

The next lines contain information about characters, one after another. The information about one character contains this character in the first place, then the width of the character, and on the next lines, the character itself. Note, that these fonts contain a column of spaces at the end of every symbol, so you don't need to add any special.

Don't worry, reading text from a file is as simple as reading from a console. You can use a scanner to do this as well. Just instead of System.`in`, pass File(pathToFont) when creating a scanner. Please, use an absolute path, to avoid problems with checking. For example, use Scanner(File("C:/fonts/roman.txt")) for the font "roman" on Windows. Or Scanner(File("/home/username/fonts/roman.txt")) on Linux/MacOS systems.

In this stage, you need to create a program that generates a name of the badge with the font "roman." It should also print the status of this person with the font "medium." The status of the person should be centered. Do not forget about cases when the length of the status is longer than the length of the name and surname! In these cases, the name and surname should be centered.
